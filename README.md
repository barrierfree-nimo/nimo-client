# ⭐️commit 규칙⭐️
  ### git commit -m “init: 초기 세팅”
  - init: 개발 환경 초기 세팅
  - feat: 새로운 기능 추가
  - design: UI를 위한 스타일링
  - fix: 버그 수정
  - docs: 문서 추가, 수정, 삭제
  - style: 코드 포맷팅(코드 순서 변경, 세미콜론 추가)
  - refactor: 코드 리팩터링
  - asset: 폰트, 이미지 추가
  - chore: 그 외 자잘한 수정


# ⭐️브랜치 전략⭐️
  ### feat/#issue_number → develop → main
  - 작업 전에 이슈 생성
  - 'feat/#이슈 번호'로 브랜치 파서 작업
  - 작업이 다 끝나면 feat 브랜치에서 develop 브랜치로 Pull Request 작성
  - 팀원 모두의 Approve를 받아야 develop 브랜치에 merge 가능
